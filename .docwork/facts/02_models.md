# GROUND TRUTH ‚Äî Mongoose Data Models (src/models/*.js)

All models use Mongoose. Cross-references are by **string `tradeRef`/`userId`**, NOT ObjectId refs.

## User (User.js) ‚Äî collection `users`
- email: String, required, unique, trimmed, lowercased
- fullName: String, required, trimmed
- password: String, required (bcrypt hash)
- createdAt: Date, default Date.now
- No `timestamps` option. No methods/hooks.

## Trade (Trade.js) ‚Äî collection `trades` ‚Äî timestamps: true
Top-level fields:
- tradeRef: String, required, unique, indexed
- originType: String, default "AUTO_GENERATED"
- tradeDate, valueDate: Date
- currentStatus: String, default "MO_PENDING"
- nextDesk: String, indexed
- amount: Number; currency: String; counterparty: String; direction: String
- entity: String; foRegion: String; product: String; tradeType: String; settlementType: String
- age: Number, default 0
- foResponseReceived: Boolean default false; cptyResponseReceived: Boolean default false
- cptyContactCount: Number default 0; foContactCount: Number default 0
- assignedTo: String default null, indexed (userId)
- isAutoGenerated: Boolean default true
- auditXml: String default null

Sub-documents:
- `truths.universal`: {amount, valueDate, currency, counterparty} ‚Äî baseline correct economics
- `truths.mo`: {amount, valueDate, currency, counterparty} ‚Äî MO desk truth (may differ from universal = FO error)
- `truths.confirmation`: {amount, valueDate, currency} ‚Äî CPTY-expected (may differ = CPTY error)
- `truths.settlement`: {amount, valueDate, currency, counterparty, beneficiaryName, beneficiaryBank, beneficiaryBIC, accountNumber, accountType, settlementMethod, correspondentBank, paymentReference, settlementDate, settlementType}
- `booking`: {amount, valueDate, currency, counterparty} ‚Äî what was booked (break = diverges from truths.mo)
- `settlementDetails`: {beneficiaryName, beneficiaryBank, beneficiaryBIC, accountNumber, accountType, currency, settlementMethod, correspondentBank, paymentReference, settlementDate, settlementType} ‚Äî system SSI shown to user
- `pendingAmendments`: Array, default []
- `amendmentHistory`: [{amendmentNumber, desk, field, oldValue, newValue, source, status, appliedAt, appliedBy}]
- `confirmationScenario`: {disputeType(default null), expectedEconomics:{amount,valueDate,currency}, evidence:[{type, provided(default false), requestedAt, receivedAt}]}
- `foEscalation`: {status(default null), escalatedAt, resolvedAt, foResponse}
- `conversation`: {status(default null), resolvedAt(default null)}

## Conversation (Conversation.js) ‚Äî collection `conversations` ‚Äî timestamps: true
- tradeRef: String required, unique, indexed
- status: String default "OPEN"
- desks: [String]
- messages: [ MessageSchema ] where MessageSchema = {sender: enum "USER"|"FO"|"COUNTERPARTY" required, body: String required, subject: String default "", timestamp: Date default Date.now, _id auto}

## FOCommunication (FOCommunication.js) ‚Äî collection `focommunications` ‚Äî timestamps: true
- tradeRef: String required, unique
- desk: String required
- openedBy: String; openedAt: Date default Date.now
- status: String default "OPEN"
- messages: [ {sender, senderRole: enum "USER"|"FO", message, timestamp default Date.now} ]

## Queue (Queue.js) ‚Äî collection `queues` ‚Äî timestamps: true
- userId: String required, unique, indexed
- desk: String required
- trades: [String] (array of tradeRef)
- sessionStart: Date default Date.now
- sessionExpiry: Date (typically sessionStart + 3h)
- isActive: Boolean default true, indexed
- lastActivity: Date default Date.now

## UserScore (UserScore.js) ‚Äî collection `userscores` ‚Äî timestamps: true
- userId: String required, unique, indexed
- points: Number default 0
- penalties: Number default 0
- tradesResolved: Number default 0
- history: [{tradeRef, action, pointsAwarded, timestamp default Date.now}]

## AuditLog (AuditLog.js) ‚Äî collection `auditlogs` ‚Äî timestamps: true
- tradeRef: String indexed
- action: String required
- userId: String indexed
- desk: String
- details: String default ""
- timestamp: Date default Date.now
- xmlContent: String default null
- isAutomated: Boolean default false

## SystemConfig (SystemConfig.js) ‚Äî collection `systemconfigs` ‚Äî timestamps: true
- key: String required, unique, indexed
- value: Mixed required
- description: String
- (seedConfig.js seeds key SETTLEMENT_INITIAL_STATE = SETTLEMENT_PENDING)

## PendingReply (PendingReply.js) ‚Äî collection `pendingreplies` ‚Äî timestamps: true
- tradeRef: String required, indexed
- replyType: String required, enum "CPTY_EMAIL"|"FO_EMAIL"|"FO_INTERNAL"
- sendAt: Date required, indexed (scheduled send time)
- subject, body, userMessage, escalationContext, desk: String
- isFinalReply: Boolean default false
- payload: Mixed

## Relationships (all by string key, no populate)
- Conversation.tradeRef 1:1 Trade.tradeRef (both unique)
- FOCommunication.tradeRef 1:1 Trade.tradeRef
- PendingReply.tradeRef N:1 Trade.tradeRef (not unique)
- Trade.assignedTo ‚Üí User._id (nullable, indexed)
- Queue.userId 1:1 User; Queue.trades[] ‚Üí Trade.tradeRef
- UserScore.userId 1:1 User; UserScore.history[].tradeRef ‚Üí Trade
- AuditLog.tradeRef/userId ‚Üí Trade/User
