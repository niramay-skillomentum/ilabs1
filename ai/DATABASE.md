# Database

## Database Engine

**MongoDB Atlas** (cloud-hosted) via **Mongoose** ODM.

Connection string: `MONGO_URI` environment variable (Atlas cluster with SSL, replica set).

Fallback: if `MONGO_URI` is missing or connection fails, the server runs in **memory-only mode** — data is not persisted.

## Collections / Models

### 1. `Trade` (`src/models/Trade.js`)

The primary entity. Each trade represents one transaction moving through the ops lifecycle.

| Field | Type | Description |
|-------|------|-------------|
| `tradeRef` | String (unique, indexed) | Primary key — e.g. `TRD_1234567890_ab12cd` |
| `originType` | String | `"AUTO_GENERATED"` (default) or `"MANUAL"` |
| `tradeDate` | Date | Date the trade was executed |
| `valueDate` | Date | Settlement/value date |
| `currentStatus` | String | Current lifecycle status (see Status Enum below) |
| `nextDesk` | String (indexed) | Which desk should handle it next |
| `amount` | Number | Notional amount |
| `currency` | String | Currency code (USD, EUR, GBP, JPY, CHF, AUD) |
| `counterparty` | String | Counterparty name (CITI, HSBC, DB, JPM, BNP, BARC, MS, UBS) |
| `direction` | String | `"BUY"` or `"SELL"` |
| `entity` | String | SGB entity (e.g., GS London, GS New York) |
| `foRegion` | String | FO region (AMER, EMEA, APAC) |
| `product` | String | FX, Equity, Derivatives, Fixed Income |
| `tradeType` | String | OTC or Listed |
| `settlementType` | String | Electronic or Bilateral |
| `age` | Number | Days since trade date |
| `truths` | Object | Desk-specific ground truths (see below) |
| `booking` | Object | What was booked in the system |
| `pendingAmendments` | Array | Pending amendment objects |
| `amendmentHistory` | Array | Full amendment audit trail |
| `confirmationScenario` | Object | Dispute type, expected economics, evidence requests |
| `foEscalation` | Object | FO escalation status, timestamps |
| `foResponseReceived` | Boolean | Whether FO has responded |
| `cptyResponseReceived` | Boolean | Whether CPTY has responded |
| `cptyContactCount` | Number | How many times CPTY has been contacted |
| `foContactCount` | Number | How many times FO has been contacted |
| `conversation` | Object | `{ status, resolvedAt }` for break resolution state |
| `assignedTo` | String (indexed) | userId of the assigned trainee |
| `isAutoGenerated` | Boolean | True if generated by system |
| `auditXml` | String | XML audit trail for auto-generated history |
| `createdAt` / `updatedAt` | Date | Mongoose timestamps |

#### Truths Sub-document

Each desk has its own "source of truth" for comparison:

```
truths: {
  universal: { amount, valueDate, currency, counterparty },
  mo:         { amount, valueDate, currency, counterparty },  // FO truth vs booking
  confirmation: { amount, valueDate, currency },              // CPTY truth (no cpty mismatch)
  settlement:   { amount, valueDate, currency, counterparty }
}
```

**MO Desk**: Compares `truths.mo` vs `booking` to detect breaks.
**Confirmation Desk**: Compares `truths.confirmation` vs trade economics.

#### Booking Sub-document
```
booking: { amount, valueDate, currency, counterparty }
```
What the ops system shows — may differ from truth if a break exists.

#### Trade Status Enum

| Status | Desk | Meaning |
|--------|------|---------|
| `MO_PENDING` | MO | Awaiting MO validation |
| `MO_BREAK_OPEN` | MO | Break raised, awaiting FO contact |
| `PENDING_FO_RESPONSE` | MO | Sent to FO, awaiting response |
| `CONFIRMATION_PENDING` | CONFIRMATION | Passed MO, awaiting CPTY confirmation |
| `LIASING_WITH_CPTY` | CONFIRMATION | Email sent to CPTY |
| `CONFIRMATION_BREAK` | CONFIRMATION | Break raised after CPTY disputes |
| `LIASING_WITH_FO` | CONFIRMATION | Escalated to FO |
| `SETTLEMENT_PENDING` | SETTLEMENT | Confirmed, awaiting settlement approval |
| `READY_FOR_APPROVAL` | SETTLEMENT | Approved, ready for final sign-off |
| `SETTLEMENT_BREAK` | SETTLEMENT | Settlement failed |

### 3. `Queue` (`src/models/Queue.js`)

Session queue per user. One active queue per user at a time.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String (unique, indexed) | The user email |
| `desk` | String | MO, CONFIRMATION, or SETTLEMENT |
| `trades` | [String] | Array of tradeRef strings |
| `sessionStart` | Date | When the session began |
| `sessionExpiry` | Date | sessionStart + 3 hours |
| `isActive` | Boolean (indexed) | Whether this session is live |
| `lastActivity` | Date | Last touch time |

---

### 4. `Conversation` (`src/models/Conversation.js`)

Email thread for a trade (CPTY/FO communication).

| Field | Type | Description |
|-------|------|-------------|
| `tradeRef` | String (unique, indexed) | Associated trade |
| `status` | String | `OPEN` or `RESOLVED` |
| `desks` | [String] | Desks that participated in this thread |
| `messages` | [MessageSchema] | Array of messages |

**MessageSchema**:
- `sender`: `"USER"`, `"FO"`, or `"COUNTERPARTY"`
- `body`: Message content
- `subject`: Email subject line
- `timestamp`: Date

---

### 5. `FOCommunication` (`src/models/FOCommunication.js`)

Internal FO escalation channel (separate from CPTY email thread).

| Field | Type | Description |
|-------|------|-------------|
| `tradeRef` | String (unique) | Associated trade |
| `desk` | String | Which desk opened the channel |
| `openedBy` | String | userId |
| `openedAt` | Date | Channel open time |
| `status` | String | `OPEN` or `CLOSED` |
| `messages` | [foMessageSchema] | Internal messages |

**foMessageSchema**:
- `sender`: Username or role name
- `senderRole`: `"USER"` or `"FO"`
- `message`: Message body
- `timestamp`: Date

## Indexes

Key indexes defined in schemas:
- `Trade.tradeRef` — unique index (primary lookup)
- `Trade.nextDesk` — for desk-based filtering
- `Trade.assignedTo` — for user session queries
- `Queue.userId` — unique (one queue per user)
- `Queue.isActive` — for finding active sessions
- `Conversation.tradeRef` — unique
- `AuditLog.tradeRef` — for trade audit lookups
- `AuditLog.userId` — for user audit lookups
- `FOCommunication.tradeRef` — unique
