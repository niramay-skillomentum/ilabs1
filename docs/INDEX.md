## What this project is (one paragraph)

`ilabs1` is a **browser-based training simulator** that teaches investment-bank **Middle-Office / Post-Trade Operations** analysts how to process trades across three desks — **MO (Middle Office)**, **Confirmation**, and **Settlement**. The system auto-generates realistic trades (some clean, some containing deliberately injected "breaks"/discrepancies), and the trainee must detect, investigate, communicate about, and resolve them. Counterparties and the front-office are simulated by **LLM-driven AI personas** (Google Gemini) with deterministic offline fallbacks, and an in-app **AI Tutor** (OpenRouter / Nvidia Nemotron) coaches the trainee. A compressed **simulation clock** turns a 9-hour trading day into a ~3-hour session. The backend is Node.js/Express + MongoDB (Mongoose) + Socket.io + Agenda; the frontend is Next.js 16 / React 19.

## Repository snapshot (as documented)

- **Backend:** Node.js (CommonJS), Express 5, Mongoose 9 (MongoDB Atlas), Socket.io 4, Agenda 5, JWT, bcryptjs.
- **AI:** Google Gemini (`gemini-2.5-flash`) for counterparty/front-office personas; OpenRouter → Nvidia Nemotron (`nvidia/nemotron-3-ultra-550b-a55b:free`) for the Tutor. (`@cerebras/cerebras_cloud_sdk` and `groq-sdk` are declared in `package.json` but **not used in code**.)
- **Frontend:** Next.js 16.2.9 (App Router), React 19.2.4, Tailwind 4, socket.io-client, react-hot-toast, react-markdown.
- **Backend default port:** `3002`. **Frontend dev port:** `3000` (docker maps frontend to `3001`).
- **Two persistence layers coexist:** Mongoose (production path) and a small **Prisma** remnant referenced only by `src/engine/settlement.js` (see [18](18_Unused_And_Dead_Code.md)).
